Tag Documentation

​

Tags are Oxyl's way of custom commands - kind of, besides the fact you have to use the tag command to call them. Tags are very customizable, giving users access to creating complex tags, or simple tags that just display text.

There are also different tag types, which go from user to global:

  • User - Anyone can create these tags and they only apply to the user whom created it
  • Channel - Anyone having ban and mute permissions can create channel-wide tags
  • Guild - Anyone having manage server permissions can create server-wide tags
  • Global - Anyone who is a bot creator can make global (bot-wide) tags

These are inherited, so if you can create a guild tag you can also create channel tag. Nobody is able to change user tags besides the user that made it themselves. Only the owner of a tag can delete the tag, unless they have a higher permission, such as someone with manage server can delete a channel tag. Guild Owners can delete guild tags, even if they did not make them.

The following are placeholders for tags, that replicate basic coding. If you need help creating a certain tag, please ask in the support server. You may test tags using oxyl tag test <content> but this does not support tag arguments

  • [ ] indicates optional arguments, empty brackets mean no arguments, or arguments will not change the outcome of the tag
  • < > indicates required arguments
  • ... indicates one or more arguments
